class flixel.system.frontEnds.LogFrontEnd

Available on all platforms

Instance Fields

var add:Dynamic

Log data to the debugger. Example: FlxG.add("Test", "1", "2", "3"); - will turn into "Test 1 2 3". Infinite amount of arguments allowed, they will be pieced together to one String.

var error:Dynamic

Add an error to the debugger. Example: FlxG.log.error("Test", "1", "2", "3"); - will turn into "[ERROR] Test 1 2 3". Infinite amount of arguments allowed, they will be pieced together to one String.

var notice:Dynamic

Add a notice to the debugger. Example: FlxG.log.notice("Test", "1", "2", "3"); - will turn into "[NOTICE] Test 1 2 3". Infinite amount of arguments allowed, they will be pieced together to one String.

var redirectTraces:Bool

Whether everything you trace() is being redirected into the log window. True by default.

var warn:Dynamic

Add a warning to the debugger. Example: FlxG.log.warn("Test", "1", "2", "3"); - will turn into "[WARNING] Test 1 2 3". Infinite amount of arguments allowed, they will be pieced together to one String.

function advanced(Data:Dynamic, Style:LogStyle, FireOnce:Bool):Void

Add an advanced log message to the debugger by also specifying a LogStyle. Backend to FlxG.log.add(), FlxG.log.warn(), FlxG.log.error() and FlxG.log.notice(). @param Data Any Data to log. @param Style The LogStyle to use, for example LogStyle.WARNING. You can also create your own by importing the LogStyle class. @param FireOnce Whether you only want to log the Data in case it hasn't been added already
